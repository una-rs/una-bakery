# una-bakery — разбор данных Unicode и запекание таблиц нормализации

Русский | [English](README.md) | [中文](README.zh.md)

----

Данный крейт предназначен для разбора данных стандарта Unicode (текущая версия - 17.0.0), и создания оптимизированных таблиц данных, используемых в алгоритме нормализации.

Библиотека является служебной, подготовленные данные используются в крейте нормализации [una-normalization](https://github.com/una-rs/una-normalization).

## Использование

```
cargo run
```

В результате будут созданы:

- Таблицы нормализации в `output/tables`.
- Статистика особенностей декомпозиции/композиции кодпоинтов в `output/stats`.

## Доверие и проверка

Этот крейт генерирует таблицы напрямую из официальных данных Unicode 17.0.0. Никаких жёстко заданных таблиц. Никакой скрытой логики.

**Для проверки:**

- Скачайте исходные файлы, указанные в [input/NOTES.md](input/NOTES.md).
- Запустите `cargo run`.
- Сравните хеш-суммы сгенерированных файлов с артефактами релиза.

## Структура

*Главные модули:*

- `src/bin` — генерация таблиц.
- `src/data` — парсинг данных из файлов, предоставленных Unicode.
- `src/normalization` — кодирование данных о нормализации кодпоинта в формат таблиц.

*Остальные модули:*

- `src/codepoint` — класс кодпоинта с его свойствами, данные получаются из `UCD/UnicodeData.txt`.
- `src/hangul` — хангыль, частный случай для кодпоинтов.
- `src/errors` — ошибки.
- `src/tests` — тесты и необязательные проверки утверждений, которые будут полезны тем, кто хочет получше разобраться в коде и стандарте.

**Входящие и исходящие данные**:

- `input` — исходные данные Unicode.
- `output/tables` — получаемые запечённые таблицы нормализации.
- `output/stats` — получаемая при кодировании статистика, дополнительная информация.
